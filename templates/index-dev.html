Your web engine in ~100 lines of code

I'm one of those that believe that Go's HTTP package is more than enough to craft a simple and lightweight web engine without any 3rd party dependencies.

We create an engine package capable of handling your future needs regarding middlewares and includes the basic thing like a logger, json parsing and decoding and a router.

No matter what is your position regarding the use of existing frameworks or routers, the covered topics can help better understand how web servers works and the life-cycle of HTTP requests -> responses.

We use Go's interface for the data package

When I initially came up with the table of content for the book I picked MongoDB as the database for no particular reason other than the ease of changing the data model.

That created some debate in the community. I decided to create the data package with both implementations of MongoDB and PostgreSQL. There's also an in-memory data store used for integration tests.

This chapter shows excellent usage of Go's interface, and hopefully, you will reuse the concept elsewhere in your future applications. We're also covering build tags to compile specific implementation of the data package.
